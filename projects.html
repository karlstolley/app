Web Application Development

A graduate course in developing applications for the Web in Ruby on Rails. Taught by Karl Stolley, Fall Semester 2012 at Illinois Institute of Technology.

Course Projects

Project One: Alpha App

Description

For the first project, you will build the alpha version of your Rails app. The app can be anything you believe to be interesting and useful. It should have only the most basic functionality and design to be viable; it's purpose is largely to help you learn to work with Rails and make mistakes. Think of it as a proof of concept, both for the app itself, and for your own initial skills in Rails development.

Requirements

  • Use Ruby 1.9.3, Rails 3.2
  • Frequent, detailed commits to your Git repository (include commands you may have run, e.g., rails generate in commit messages, when applicable) and frequent pushes to your GitHub remote repository
  • Do development work on a dev branch in Git; merge only the best, functional, test-passing work over to the master branch.
  • Use bootstrap-sass-rails or another gem of your choice for building over Twitter Bootstrap as a baseline for your appâ€™s design
  • Strive for at least two database tables, associated in some way
  • Develop a suite of functional and unit tests for your application

Deliverables

  1. A functional Rails app that can be racked up and run locally
  2. The GitHub repository for the app (link will be emailed to the instructor)
  3. A 5-minute presentation of your app on October 10
  4. A project memo, submitted as an email to the instructor the night the app is due (October 10)
Project Two: Beta App

Description

For the beta app, you will either begin again with the app created for Project One, or rework/abandon your idea for a brand-new app. Requirements and deliverables will be similar to those of Project One; however, you will more fully flesh out the design of your app, both functionally and visually. You will also deploy your app to a public-facing server and be responsible for some means of user/administrative authentication.

Deliverables

  • A functional Rails app that can be racked up and run locally, and deployed long enough for proof-of-concept to a public facing server (e.g., on Heroku)
  • Three types/areas of functionality that you intend to add, proposed to the instructor by October 23 (hint: score Rails Recipes for inspiration and ideas, if you're coming up short).
  • Some means of user/administrator authentication; can be very basic, or can involve a Gem
  • The GitHub repository for the app (link will be emailed to the instructor)
  • A ten-minute presentation of your app on December 5
  • A project memo, submitted as an email to the instructor the night the app is due (December 5)
